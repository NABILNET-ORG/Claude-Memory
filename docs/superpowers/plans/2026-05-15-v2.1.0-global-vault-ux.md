# v2.1.0 GLOBAL Vault UX — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a browse-only `list_global_patterns` MCP tool that lets the agent and the user enumerate the reserved `'GLOBAL'` Knowledge Vault by Sovereign-Taxonomy attribute, with tiered output and zero embedding cost. Surface it via an extended `init_project` capabilities header so a fresh boot discovers the tool without reading docs.

**Architecture:** A new tool handler (`src/tools/list-global-patterns.ts`) executes a pure SQL `SELECT` against `memory_chunks` scoped to `project_id='GLOBAL'`, with JSONB-containment filtering through the existing GIN(`jsonb_path_ops`) index. The Zod schema for `metadata_filter` is **hoisted once** into `src/tools/shared-schemas.ts` and reused by both `search_memory` and `list_global_patterns` — strict DRY. The `init_project` capabilities builder is extracted to a pure function so the shape contract becomes unit-testable. No new dependencies, no new indexes, no new migrations.

**Tech Stack:** TypeScript + `@modelcontextprotocol/sdk` (MCP server); `zod` (input validation); `pg` + `@supabase/supabase-js` (DB access); `node:test` + `node:assert/strict` via `tsx` (test runtime). All existing — no additions.

**Spec contract:** [ARCHITECTURE.md §4.3.1](../../ARCHITECTURE.md) — the 10 ACs in that section are the executable contract this plan satisfies. The spec defined a 6-commit sequence; the DRY-reuse constraint expands that to **7 commits** (one new refactor commit upstream of the feature commits to hoist the shared Zod schema).

```typescript
export async function listGlobalPatterns(
  args: ListGlobalPatternsArgs,
): Promise<ListGlobalPatternsResponse> {
  const limit = Math.min(args.limit ?? DEFAULT_LIMIT, MAX_LIMIT);
  const offset = args.offset ?? 0;
  const includeContent = args.include_content ?? false;

  let query = supabase
    .from("memory_chunks")
    .select("id, content, metadata, file_origin, created_at", { count: "exact" })
    .eq("project_id", "GLOBAL")
    .order("created_at", { ascending: false })
    .order("id", { ascending: false })
    .range(offset, offset + limit - 1);

  if (args.metadata_filter && Object.keys(args.metadata_filter).length > 0) {
    // @>-style JSONB containment: Supabase JS exposes this via the
    // `contains` operator. Hits the existing GIN(jsonb_path_ops) index on
    // memory_chunks.metadata.
    query = query.contains("metadata", args.metadata_filter);
  }

  const { data, error, count } = await query;
  if (error) {
    throw new Error(`list_global_patterns SELECT failed: ${error.message}`);
  }

  const rows = (data ?? []).map((r): ListGlobalPatternsRow => {
    const metadata = (r.metadata ?? {}) as Record<string, unknown>;
    const content = (r.content ?? "") as string;
    const preview =
      content.length <= 120 ? content : `${content.slice(0, 120)}…`;
    const row: ListGlobalPatternsRow = {
      id: r.id as number,
      type: typeof metadata.type === "string" ? metadata.type : null,
      global_rationale:
        typeof metadata.global_rationale === "string"
          ? metadata.global_rationale
          : null,
      created_at: r.created_at as string,
      file_origin: (r.file_origin ?? "") as string,
      content_preview: preview,
    };
    if (includeContent) {
      row.content = content;
    }
    return row;
  });

  const total = count ?? rows.length;
  const summary =
    total === 0
      ? "GLOBAL vault is empty."
      : `Returned ${rows.length} of ${total} GLOBAL row(s).`;

  return {
    project_id: "GLOBAL",
    count: total,
    results: rows,
    limit,
```

## Glossary

- **`metadata_filter`**: a `Record<string, unknown>` passed verbatim to Postgres `metadata @> $filter`. Matches JSONB containment. Uses the existing GIN(`jsonb_path_ops`) index on `memory_chunks.metadata`.
- **Tiered output**: default response per row omits the `content` field and includes `content_preview` (≤120 chars + ellipsis); `include_content:true` adds the full `content` field.
- **`GLOBAL` vault**: reserved literal `project_id='GLOBAL'` storing universal truths that pass the Cross-Project Test. Distinct from the slugified-cwd project_id of any concrete repo.
- **Capabilities Header**: the `capabilities` block in `init_project`'s response. Tells the agent the protocol version, the GLOBAL vault location + browse tool, the Sovereign Taxonomy enum, and a list of context-gathering hint exemplars.
